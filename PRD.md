# team_calender PRD

## 1. 배경 및 목적

대행사 환경에서는 팀원들의 재택 여부, 휴가 일정, 회사 공통 일정을 한눈에 파악하기 어렵다. 개별 메신저나 구두로 공유되던 일정 정보를 팀 단위 캘린더로 통합하여, 불필요한 커뮤니케이션 비용을 줄이고 팀원 간 일정 가시성을 높인다.

## 2. 목표 및 성공 지표

| 목표 | 지표 |
|------|------|
| 일정 공유 커뮤니케이션 감소 | 월간 "오늘 재택이에요?" 메시지 50% 감소 |
| 캘린더 활성 사용 | 주간 활성 사용자(WAU) 전체 직원의 80% 이상 |
| 휴가 처리 속도 향상 | 휴가 신청 → 확정 소요 시간 1영업일 이내 |

## 3. 타겟 사용자

- **주 사용자**: 대행사 전체 직원 (팀원)
- **관리자**: 팀장 / HR 담당자 (휴가 승인, 회사 일정 등록 권한)
- **사용 환경**: 사무실 PC, 재택 시 개인 노트북 (웹 브라우저)

## 4. 핵심 기능 정의

### 4-1. 인증
- 이메일 + 비밀번호 로그인
- 팀 단위 계정 분리 (타 팀 캘린더 열람 불가)

### 4-2. 팀 캘린더
- 팀별 월간 캘린더 뷰
- 회사 공통 일정 등록 및 표시 (공휴일, 전사 행사 등)
- 당일 재택 여부 표시 (출근 / 재택 / 외근)

### 4-3. 휴가 관리
- 휴가 신청 (예정 상태)
- 관리자 승인 후 확정 처리
- 확정 시 구글 캘린더 자동 연동 + 팀 메일 자동 발송

## 5. 사용자 흐름 (User Flow)

```
[로그인]
  └─ 이메일+비밀번호 입력
       └─ 내 팀 캘린더 메인 화면

[일정 확인]
  └─ 월간 캘린더에서 팀원별 재택/휴가 상태 확인

[재택 등록]
  └─ 오늘 날짜 클릭 → 출근/재택/외근 선택 → 저장

[휴가 신청]
  └─ 날짜 선택 → 휴가 신청 → 관리자 승인 대기
       └─ 승인 완료 → 구글 캘린더 연동 + 팀 메일 자동 발송

[회사 일정 등록] (관리자)
  └─ 날짜 선택 → 일정명 입력 → 전체 팀 캘린더에 표시
```
